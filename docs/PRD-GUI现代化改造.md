# PRD: Bilibili AI Client GUI 现代化改造

## 1. 背景与现状

当前使用 **tkinter** 作为 GUI 框架，界面风格停留在 Windows 98/XP 时代。tkinter 存在以下痛点：

- 无原生主题支持，视觉风格陈旧
- 布局系统原始（pack/grid/place），复杂 UI 维护困难
- 无现代控件（表格、树形视图、富文本等）
- 高 DPI 缩放支持差
- 与 asyncio 事件循环集成需 hack

## 2. 目标

为 Bilibili AI Client 替换一个**现代化、可维护、美观**的 GUI 框架，保持现有全部功能，同时为新功能（如仪表盘、实时日志等）提供扩展基础。

## 3. 候选方案调研

### 方案一：CustomTkinter — 最小迁移成本

- 基于 tkinter 的现代化主题包装
- 圆角控件、明暗主题
- **优势**：几乎不改架构，数小时即可完成迁移
- **劣势**：底层仍是 tkinter，上限低；复杂布局依然难受
- **打包后体积**：~30MB（不含额外依赖）
- **适用场景**：希望快速改善外观，不追求架构升级

### 方案二：PySide6 — 专业级桌面框架

- Qt6 官方 Python 绑定，LGPL 许可证
- Qt Designer 可视化设计，QSS 样式表
- Model/View 架构处理大量数据，QThread 线程模型
- **优势**：功能最全，生态成熟，专业控件，长期可维护
- **劣势**：学习曲线陡，打包后体积较大（60-80MB）
- **适用场景**：希望构建专业级桌面应用，长期投入

### 方案三：Flet — 现代化响应式 UI（Flutter 渲染）

- 用 Python 写 Flutter UI，Material Design 设计语言
- 响应式布局，动画过渡，跨平台（桌面+Web+移动）
- **优势**：UI 最现代化，开发效率高，视觉效果一流
- **劣势**：较新（生态不如 Qt），Flutter 引擎 ~50MB，异步模型与 sync 代码整合需注意
- **适用场景**：追求极致视觉体验，愿意接受较新框架

### 方案四：Dear PyGui — GPU 加速工具型 UI

- 即时模式 GUI，GPU 渲染，高性能
- Pythonic API，自带暗色主题
- **优势**：上手快，性能好，适合内部工具
- **劣势**：非原生外观，控件不如 Qt 丰富，不适合传统桌面应用范式
- **适用场景**：数据仪表盘、内部工具

## 4. 推荐方案

**首选：PySide6** — 理由如下：

1. **许可证友好**：LGPL 允许闭源商业分发，无需购买商业许可
2. **工程成熟**：Qt 25 年历史，PySide 官方维护，社区活跃
3. **工具链完善**：Qt Designer（拖拽设计）、QSS（样式表）、QThread（线程管理）
4. **与现有代码兼容**：已有 PyInstaller 打包流程可复用，PySide6 的 PyInstaller 支持已稳定
5. **功能对等或超越**：TabWidget（替代 Notebook）、ListWidget/TableView（替代 Listbox）、QSS 主题（替代 tkinter 的主题挣扎）
6. **渐进式迁移**：可先用 Qt Widgets 保持传统布局，未来再引入 QML/QtQuick 实现高级动画

**备选：Flet** — 如果团队更看重视觉效果且愿意拥抱较新技术栈，Flet 的 Material Design 响应式 UI 能提供最佳用户体验。

## 6. 技术约束

- **Python >= 3.11**：保持现有要求
- **PyInstaller 打包**：必须可打包为单个 .exe
- **asyncio 集成**：GUI 需与现有的 asyncio 事件循环（MCP server）共存
- **加密 Cookie**：GUI 中 Cookie 显示为 `********`，不可明文暴露
- **SQLite 数据库**：数据层不变，仅替换 UI
- **现有回调体系**：`on_openclaw_complete`、`process_new_message` 等回调保持兼容

## 7. 非功能需求

| 指标 | 目标 |
|------|------|
| 启动时间 | < 3 秒（含服务初始化） |
| 内存占用 | < 200 MB |

## 9. 风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| PySide6 + PyInstaller 兼容问题 | 低 | 高 | 使用官方推荐配置，提前验证打包 |
| asyncio 事件循环集成困难 | 中 | 中 | 使用 `qasync` 库桥接 Qt 和 asyncio |
| 现有 tkinter 代码深度耦合 | 低 | 低 | UI 层已独立于 `gui/` 模块，替换不影响后端 |
| 学习曲线导致开发周期延长 | 中 | 中 | 优先使用 Qt Designer 加速，参考 pythonguis.com 教程 |
| 用户对新 UI 不适应 | 低 | 低 | 保持功能布局与旧版一致，仅视觉升级 |
